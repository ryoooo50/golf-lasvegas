- [x] 1. P0: ゲーム計算ロジックのバグ修正（テストファースト）
- [x] 1.1 既存テストを仕様準拠の期待値に書き直す
  - バーディーは倍率に影響しない（Flipのみ）ことを検証するテストに修正する
  - キャリーオーバーがx2→x4→x6と+2ずつ増加することを検証するテストを修正・追加する
  - プッシュとキャリーオーバーが同時発生した場合は倍率を「加算」（乗算でない）することを検証するテストを追加する
  - 両チームバーディー時に両方Flipし倍率変化なしであることを検証するテストを追加する
  - drawが3連続した場合のnextMultiplier推移（1→2→4→6）を検証するテストを追加する
  - _Requirements: 1.1-1.4, 2.1-2.6, 3.1-3.5, 10.1-10.5_

- [x] 1.2 キャリーオーバー倍率の計算ロジックを仕様通りに修正する
  - 引き分け時のnextMultiplierを「現在が1倍なら2倍、2倍以上なら現在+2倍」に修正する（現在は+1）
  - 勝敗確定時にnextMultiplierを1にリセットする処理は維持する
  - テストが全て通ることを確認する
  - _Requirements: 1.1-1.4_

- [x] 1.3 プッシュ倍率とバーディーの役割を仕様通りに分離する
  - プッシュ倍率を「0人なら×1、n人なら×(2n)」として計算する
  - バーディーの有無を倍率計算から除外する（バーディーはFlipのみに影響する）
  - 最終ポイントを「差分 × 有効倍率」で算出し、プッシュとキャリーオーバーが両方ある場合は有効倍率を `PushMultiplier + CarryOverMultiplier` とする
  - リアルタイム倍率表示用のヘルパー関数も同じロジックに統一する
  - テストが全て通ることを確認する
  - _Requirements: 2.1-2.6, 3.1-3.5_

- [x] 1.4 計算結果の詳細内訳データを型定義とロジックに追加する
  - 各チームの「連結前スコア・連結後スコア・Flipしたか・Flip後スコア」を格納する型を定義する
  - 「差分・プッシュ倍率・キャリーオーバー倍率・最終倍率・最終ポイント・引き分けか」を格納する型を定義する
  - ホール計算結果（HoleResult）に詳細内訳フィールドを追加する
  - calculateHoleResultが詳細内訳を生成して返すように修正する
  - 既存のsavedRoundsデータ（内訳なし）を参照するコードはnullガードを追加する
  - _Requirements: 6.1-6.4_

- [x] 1.5 Hole 9・18の非仕様プッシュ強制使用ロジックを削除する
